Welcome to STAP Research Group!

The STAP Research Group does research in many areas of software engineering, including:

  • Design languages and tool support
  • Static and dynamic program analysis
  • Aspect-oriented software development
  • Various aspects of analysis
  • Testing and maintenance of software systems

The STAP Research Group's overall goal is to improve software development practice and provide a variety of tools that will lead to better understanding and maintenance of software systems.

This site contains a brief introduction of our research group, PDF files of our publications, PowerPoint slides from various talks, a list of group members, and even some pictures. You can learn more about our group by following the links on the left. If you have any questions regarding the group, please email any one of us.

Research Topics

Program Repair

Automated Program Repair approaches automatically or semi-automatically modify buggy program to satisfy given correctness criteria.

Programming with Big Code

Just like huge amounts of data on the web enabled Big Data applications, now large repositories of programs (e.g. open source code on GitHub) enable a new class of applications that leverage these repositories of "Big Code". Using Big Code means to automatically learn from existing code in order to solve software engineering tasks such as predicting software bugs, predicting program behavior, or automatically generating new code.

Automatic Programming

Automatic Programming is the process of automatically generating programs by a computer based on an informal (or formal) specification.


Research Approaches

Machine Learning

Machine Learning is a type of artificial intelligence (AI) that provides computers with the ability to learn without being explicitly programmed.

Program Analysis

Program Analysis is the process of automatically analyzing the behavior of computer programs regarding a property such as correctness, robustness, safety and liveness.

Software Testing

Software Testing is an investigation conducted to provide stakeholders with information about the quality of the product or service under test. Test techniques include the process of executing a program or application with the intent of finding software bugs (errors), and to verify that the software product is fit for use.
